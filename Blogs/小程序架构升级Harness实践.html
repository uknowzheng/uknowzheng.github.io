Harness Engineering介绍

Harness Engineering(驾驭工程)是什么

提示词工程 → 上下文工程 → 驾驭工程

怎么让模型稳定、可靠、不失控 核心隐喻：

  • 大模型 = CPU
  • Agent = 操作系统，CPU 执行指令，OS 管理内存、调度进程、处理错误

行业洞察：

“The model is commodity. The harness is moat.”（模型是商品，Harness 才是护城河）

两个团队使用相同模型，任务完成率可能相差 40 个百分点（60% vs 98%），差异完全来自 Harness 质量。

## 技术栈
- **框架**: UniApp 2.0.2 + Vue 2.6.10 + Composition API (@vue/composition-api)
- **构建**: Vue CLI 4.x + Webpack
- **包管理**: pnpm + Node.js 22
- **代码规范**: ESLint 6.8.0 + Prettier 2.8.8 + Husky pre-commit
- **状态管理**: Vuex 3.2.0、Pinia 2.1.7
- **测试**: Jest + @vue/test-utils + @vue/cli-plugin-unit-jest
