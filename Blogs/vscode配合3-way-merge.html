vscode配合3 way merge

Apr 24, 20264 min read

因为传统的编辑器是不具备代码冲突处理的逻辑的，而且git默认也是可以指定当冲突发生的时候去指定mergetool对代码冲突进行处理，默认是2 way merge的这种方式（这种方式很难看出代码整体结构差异，会发生遗漏代码的情况）

为什么需要3 way merge？因为传统的merge其实是一种2 way merge的形式，这种方式merge最直观的表现如下：

相对于2 way merge来说3 way merge会更加直观一些，比如：

jetbrain全家桶ide基本集成了自家的mergetool且支持3 way merge，这种是webstorm在你进行分支合并的时候，底层默认使用了git mergetool -t intellij 进行拉起的。

对于vscode在解决代码冲突的场景下当前是不支持使用3 way merge的，社区的issue一直打开着：https://github.com/microsoft/vscode/issues/37350

所以需要一些辅助的第三方软件来帮助git发生冲突时能够通过调用mergetool进行3 way merge对合并冲突的文件进行处理。以下是支持3 way merge的工具：

  • KDiff3
  • DiffMerge
  • P4Merge
  • Meld
  • Beyond Compare Pro.

其中有很多软件是要进行收费处理的。

这里推荐使用meld免费且开源，具体下载可以查看官网推荐地址：http://meldmerge.org/

下载好后，拖拽完成安装：

接下来打开mac终端，指定git merge tool的配置：

vim ~/.gitconfig

新增如下配置项目（根据实际情况配置）：

[merge]
tool = meld
[mergetool]
prompt = false
[mergetool “meld”]
trustExitCode = true
cmd = open -W -a Meld —args —auto-merge “PWD/LOCAL” “PWD/BASE” “PWD/REMOTE” —output=“PWD/MERGED”

完成后，当你merge 分支冲突的时候：

上述因为我是设置成全局默认，所以编辑了gitconfig文件。实际你也可以：git mergetool -t meld指定工具为meld。

上述命令行运行完成后，终端输出如下：

会拉起meld处理冲突的文件，这个时候只需要简单点点点就可以合并代码了：

当处理完成后，保存中间这个文件。然后选择左上角红色x号关闭即可表示处理完当前冲突文件。下一个文件会开启一个新的meld窗口，再进行同样操作，直至最后一个冲突文件处理完成。

反复处理保存后，就能够完成冲突处理。
