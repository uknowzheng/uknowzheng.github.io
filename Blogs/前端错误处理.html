try catch finally

在 try 和 catch 之后无论有无异常（return或者throw Error）都会执行finally里面的逻辑。

JS异常捕获crash或者纯报错

window.onerror与window.addEventListener(‘error’)捕获js运行时错误window.onerror含有详细的error堆栈信息，存在error.stack中，所以我们选择使用onerror的方式对js运行时错误进行捕获。所以不选择window.addEventListener(‘error’)捕获

异步任务报错捕获

由于async await函数和promise可能比较多，项目中，为了防止没有捕获的promise出现，我们可以使用 window.addEventListener(‘unhandledrejection’)捕获到没有处理错误的promise

React16中注意的问题

在 react 16 以后，任何未被错误边界捕获的错误将会导致整个 React 组件树被卸载。可以使用componentDidCatch和getDerivedStateFromError回调函数进行捕获处理。

体验问题（PerformanceObserverAPI）

  • 性能时间问题：基于NavigationTiming实现，用于定位首次开启白屏时常体验问题
  • 卡顿：当浏览器内核UI线程卡顿时间超过50ms则记一次卡顿异常，技术上通过Long Task API实现。
  • 滚动卡顿：以标准的PerformanceObserverAPI的方式将这2个指标暴露给了TES，TES会通过该API自动监听用户每一次滑动到页面开始滚动之间的延迟，当延迟超过100ms时记一次错误
  • 动画卡顿：requestAnimationFrame
